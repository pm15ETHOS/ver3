## 섹션 1 — 상단 KPI 요약 카드

### 표시 항목

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  전체 이수율  │  │  평균 만족도  │  │ 인기 시나리오 │
│   68%        │  │   4.2 / 5    │  │ AI 채용 편향  │
│ (34 / 50명)  │  │              │  │ 완료율 82%   │
└──────────────┘  └──────────────┘  └──────────────┘
```

### 데이터 정의

| 항목 | 계산 방식 |
|------|-----------|
| 전체 이수율 | 이수 완료 인원 / 전체 대상 인원 × 100 |
| 평균 만족도 | Q1(전체 만족도) 응답 평균값 |
| 인기 시나리오 | 5개 시나리오 중 이수 완료율이 가장 높은 시나리오명 |

## 섹션 2 — 팀별 이수 현황

### 표시 형식

가로 막대 그래프 (Horizontal Bar Chart)
각 팀별로 한 줄씩 표시.

```
개발 A팀   ████████░░  60%  (3 / 5명)
개발 B팀   ██████████  100% (5 / 5명)
데이터팀   ██████░░░░  60%  (3 / 5명)
기획팀     ████░░░░░░  40%  (2 / 5명)
QA팀       ████████░░  80%  (4 / 5명)
```

### 색상 기준

| 이수율 | 색상 |
|--------|------|
| 80% 이상 | 초록 |
| 50~79% | 노랑 |
| 50% 미만 | 빨강 |

### 데이터 정의

| 항목 | 설명 |
|------|------|
| 팀명 | 사전 등록된 팀 단위 (개인 식별 정보 없음) |
| 이수 완료 | 시뮬레이션 1개 이상 설득 성공 + 만족도 제출 완료 |
| 미이수 | 미시작 또는 진행 중 상태 |

## 섹션 3 — 시나리오별 완료 현황

### 표시 형식

시나리오별로 누적 가로 막대 그래프 (Stacked Bar Chart)
각 막대는 3가지 상태로 구성.

```
AI 채용 편향          [✅ 완료 55%][🔄 재도전 후 완료 27%][🎲 시나리오 전환 18%]
내부 코드 GPT 입력    [✅ 완료 40%][🔄 재도전 후 완료 35%][🎲 시나리오 전환 25%]
Copilot 코드 무검증   [✅ 완료 50%][🔄 재도전 후 완료 30%][🎲 시나리오 전환 20%]
AI 오류 무시 출시     [✅ 완료 30%][🔄 재도전 후 완료 25%][🎲 시나리오 전환 45%]
추천 알고리즘 차별    [✅ 완료 45%][🔄 재도전 후 완료 33%][🎲 시나리오 전환 22%]
```

### 상태 정의

| 상태 | 아이콘 | 색상 | 정의 |
|------|--------|------|------|
| 이수 완료 | ✅ | 초록 | 1회 시도 만에 설득 성공 |
| 재도전 후 완료 | 🔄 | 파랑 | 실패 후 같은 시나리오 재시도하여 최종 성공 |
| 시나리오 전환 | 🎲 | 회색 | 실패 후 다른 시나리오로 전환 (해당 시나리오 미완료 처리) |

### 관리자 인사이트 텍스트 (자동 생성)

시나리오 전환율이 40% 이상인 시나리오에는 아래 문구를 자동 표시.

```
⚠️ 이 시나리오는 전환율이 높습니다. 난이도 조정 또는 힌트 보강을 검토해보세요.
```

## 섹션 4 — 만족도 평균

### 표시 형식

5개 문항별 전사 평균 점수를 가로 바 또는 수치로 표시.

```
Q1 전체 만족도       ⭐⭐⭐⭐☆  4.2점
Q2 교육 영상 유익성  ⭐⭐⭐⭐☆  3.9점
Q3 난이도 적절성     ⭐⭐⭐☆☆  3.2점
Q4 학습 효과         ⭐⭐⭐⭐☆  4.1점
Q5 재참여 의향       ⭐⭐⭐⭐☆  4.0점
```

### 데이터 정의

| 항목 | 계산 방식 |
|------|-----------|
| 각 문항 평균 | 해당 문항 전체 응답값 합계 / 응답 인원 수 |
| 표시 단위 | 소수점 첫째 자리까지 |
| 집계 범위 | 만족도 제출 완료자 전체 (팀 구분 없이 전사 평균) |

### 주의

- 개인별 응답 데이터는 저장하지 않음
- 집계 평균값만 대시보드에 표시
- 응답 인원이 5명 미만인 팀은 팀별 분리 표시 안 함 (전사 합산만 표시)

## 섹션 5 — 설득 유형 분포

### 표시 형식

도넛 차트 (Donut Chart) — 전사 기준

```
🔍 증거 제시형      35%
⚠️ 리스크 경고형    25%
💬 질문 유도형      15%
🤝 공감 유도형      12%
⚖️ 원칙 강조형      8%
💡 관계 기반형      5%
```

### 데이터 정의

| 항목 | 계산 방식 |
|------|-----------|
| 각 유형 비율 | 해당 유형 판정 인원 / 이수 완료 전체 인원 × 100 |
| 집계 기준 | 이수 완료자 중 결과 화면까지 진행한 인원 |
| 표시 단위 | % (소수점 반올림) |

### 관리자 인사이트 텍스트 (자동 생성)

가장 비율이 낮은 유형에 대해 아래 문구 자동 표시.

```
💡 [관계 기반형] 설득 스타일 비율이 낮습니다.
   팀워크와 신뢰 기반의 소통 역량 강화 교육을 고려해보세요.
```

## 데이터 수집 및 전송 흐름

```
[사용자 시뮬레이션 완료]
        ↓
[만족도 테스트 제출]
        ↓
[백엔드(FastAPI)로 전송하는 데이터]

{
  "team": "개발 A팀",               // 팀명 (개인 식별 정보 없음)
  "scenario_id": "scenario_01",     // 완료한 시나리오 ID
  "completion_status": "first_try", // "first_try" | "retry" | "switched"
  "satisfaction": {
    "q1": 4,
    "q2": 5,
    "q3": 3,
    "q4": 4,
    "q5": 5
  },
  "persuasion_type": "증거 제시형"  // GPT 분석 결과
}

※ 개인 식별 정보(이름, 사번, 이메일 등) 일절 포함하지 않음
※ 개선점 서술형 응답은 별도 익명 텍스트로만 저장 (팀명도 미포함)
```

---

## completion_status 값 정의

| 값 | 의미 |
|----|------|
| `first_try` | 1회 시도 만에 설득 성공 → ✅ 이수 완료 |
| `retry` | 실패 후 동일 시나리오 재도전하여 성공 → 🔄 재도전 후 완료 |
| `switched` | 실패 후 다른 시나리오로 전환 → 🎲 시나리오 전환 (해당 시나리오 미완료) |
