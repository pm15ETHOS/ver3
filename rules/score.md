# 만족도 테스트 + 결과 화면 분석 프롬프트

---

## 1. 만족도 테스트 UI 구성

설득 완료 후 아래 5개 질문을 순서대로 사용자에게 제시한다.
각 질문은 ⭐ 1~5점 척도로 응답받는다.
모든 응답 완료 후 "결과 보기" 버튼을 통해 결과 화면으로 이동한다.
개선점 서술형 입력은 선택사항으로, "결과 보기" 버튼 하단에 별도 제공한다.

---

### 만족도 질문 5개

| 번호 | 질문 | 척도 안내 |
|------|------|-----------|
| Q1 | 이번 시뮬레이션 경험이 전반적으로 만족스러우셨나요? | 1=매우 불만족 ~ 5=매우 만족 |
| Q2 | 사전 교육 영상이 시뮬레이션을 이해하는 데 도움이 됐나요? | 1=전혀 도움 안 됨 ~ 5=매우 도움됨 |
| Q3 | 시나리오의 난이도는 적절했나요? | 1=너무 쉬움 ~ 5=너무 어려움 |
| Q4 | 이 시뮬레이션을 통해 AI 윤리에 대해 새롭게 배운 점이 있었나요? | 1=전혀 없음 ~ 5=매우 많음 |
| Q5 | 다른 시나리오도 도전해보고 싶으신가요? | 1=전혀 없음 ~ 5=매우 그렇다 |

---

### 개선점 서술형 (선택)

```
💬 이번 시뮬레이션에서 아쉬웠던 점이나 개선됐으면 하는 점을 자유롭게 적어주세요.
(선택사항 — 작성하지 않아도 결과를 확인할 수 있어요)

[                                        ]

[ 결과 보기 →]
```

## 2. 결과 화면 분석 시스템 프롬프트 (System Prompt)

```
당신은 AI 윤리 교육 시뮬레이션의 결과 분석가입니다.
사용자와 페르소나의 전체 대화 기록을 분석하여 아래 JSON 형식으로만 응답하세요.
JSON 외의 텍스트, 마크다운 코드블록, 설명은 절대 포함하지 마세요.
```

---

## 3. 결과 화면 분석 유저 프롬프트 (User Prompt)

```
아래는 AI 윤리 교육 시뮬레이션의 전체 대화 기록과 만족도 응답입니다.

[시나리오]
{{SCENARIO_NAME}}

[대화 기록]
{{CONVERSATION_HISTORY}}

[만족도 응답]
Q1 전체 만족도: {{Q1_SCORE}}점
Q2 교육 영상 유익성: {{Q2_SCORE}}점
Q3 난이도 적절성: {{Q3_SCORE}}점
Q4 학습 효과: {{Q4_SCORE}}점
Q5 재참여 의향: {{Q5_SCORE}}점

위 내용을 분석하여 다음 JSON 형식으로 응답하세요.

{
  "highlight": {
    "quote": "사용자의 발언 중 페르소나의 태도를 가장 크게 바꾼 결정적 한 마디를 원문 그대로 발췌 (1~2문장)",
    "reason": "왜 이 발언이 결정적이었는지 2~3문장으로 설명. 어떤 윤리적 논점을 건드렸는지 포함."
  },
  "persuasion_type": {
    "label": "설득 유형 이름 (아래 6가지 중 하나 선택)",
    "emoji": "해당 유형의 이모지",
    "description": "이 사용자의 설득 방식을 2~3문장으로 구체적으로 설명. 대화에서 실제로 쓴 전략을 언급할 것."
  },
  "score_comment": "만족도 점수 전체를 종합하여 1~2문장의 격려 또는 피드백 멘트. 딱딱하지 않고 따뜻한 톤으로."
}

---

[설득 유형 6가지 — label과 emoji는 아래 중 하나만 사용]

1. label: "증거 제시형", emoji: "🔍"
   → 실제 사례, 데이터, 통계를 활용해 논리적으로 설득하는 스타일

2. label: "공감 유도형", emoji: "🤝"
   → 상대방의 입장을 먼저 이해하고 감정적 공감을 통해 설득하는 스타일

3. label: "질문 유도형", emoji: "💬"
   → 직접 주장하기보다 질문을 던져 상대가 스스로 문제를 깨닫게 하는 스타일

4. label: "원칙 강조형", emoji: "⚖️"
   → 윤리 원칙, 규정, 법적 기준을 중심으로 설득하는 스타일

5. label: "리스크 경고형", emoji: "⚠️"
   → 발생 가능한 피해, 법적 책임, 기업 리스크를 구체적으로 제시하는 스타일

6. label: "관계 기반형", emoji: "💡"
   → 팀워크, 신뢰, 동료로서의 책임감을 강조하며 설득하는 스타일
```

## 5. API 호출 흐름 요약

```
1. 시뮬레이션 설득 완료
        ↓
2. 만족도 테스트 5문항 응답 수집
        ↓
3. (선택) 개선점 서술형 입력
        ↓
4. [결과 보기] 버튼 클릭
        ↓
5. GPT API 호출
   - System Prompt: 결과 분석가 지침
   - User Prompt: 대화기록 + 만족도 점수 삽입
        ↓
6. JSON 파싱 후 결과 화면 렌더링
   - highlight.quote / reason
   - persuasion_type.label / emoji / description
   - score_comment
        ↓
7. 만족도 데이터 → HR 관리자 대시보드 집계 전송
   (개인 식별 정보 제외, 집계 데이터만)
```
