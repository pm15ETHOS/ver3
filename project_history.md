# ETHOS - AI 윤리 교육 챗봇 개발 기록

## 프로젝트 개요
ETHOS는 AI 실무자 및 관련 부서를 대상으로 하는 **AI 윤리 교육 시뮬레이션 시스템**입니다. `Streamlit`을 활용하여 웹 애플리케이션으로 구축되었으며, 사용자는 5가지의 실무 기반 윤리 딜레마 시나리오에서 AI 페르소나(`gpt-4o-mini` 모델 연동)와 대화하며 설득형 학습을 진행합니다.

---

## 📅 주요 개발 내역 요약

### 1. 프로젝트 초기 계획 및 구조 설계
- 제공된 룰 문서[hrdashboard.md], [persona.md], [score.md]와 레퍼런스[app_claude_vibe.py], [app_gpt_vibe.py]를 분석하여 전체 앱 아키텍처를 설계.
- 사용자(User)와 관리자(Admin) 모드를 분리하고, 5단계의 커리큘럼(Stage 0 ~ 4)을 기획.

### 2. UI/UX 및 공통 컴포넌트 구현
- **디자인 시스템**: 와이어프레임 구조를 반영하여 청록색(`teal`, `#009b84`)을 메인 테마 색상으로 지정.
- **커스텀 CSS**: 버튼(`stButton`), 진행 바(`stProgress`), 정보 카드(`custom-card`) 등 전반적인 UI 요소 스타일링.
- **세션 상태(Session State)**: [init_session_state()] 함수를 통해 사용자 로그인 상태, 현재 단계(`current_stage`), 비디오 인덱스(`video_index`), API 키(`api_key`)를 전역으로 관리.
- **레이아웃**: 좌측 사이드바[render_sidebar]에 진행률과 목차를 표시하고, 상단바[render_topbar]에 유저 정보와 모드 전환/로그아웃 버튼 배치.

### 3. Stage 0. 프로그램 소개 및 로그인 (Intro)
- 탭(`st.tabs`)을 활용하여 사용자 로그인과 관리자 로그인을 분리.
- 로그인 완료 시 "ETHOS 프로그램" 소개 화면이 렌더링되며, 4개의 정보 카드(교육 커리큘럼, 인터랙티브 학습, 시나리오, 학습 관리)와 전체 학습 플로우가 표시됨.

### 4. Stage 1. AI 윤리 교육 영상 (Video Player)
- 총 6개의 윤리 교육 주제("AI 윤리 개념", "AI 프라이버시", "정보 보호", "할루시네이션", "알고리즘 편향", "AI 사용 명시")를 순서대로 시청할 수 있도록 `video_index` 로직 구현.
- 사용자가 확인한 영상에 따라 사이드바 내비게이션의 텍스트와 초록색 하이라이트 위치가 동적으로 변경됨.
- 각 영상 시청 완료 후 나타나는 `[바로 다음으로 이동]` 버튼의 레이아웃과 디자인을 와이어프레임 스펙에 맞게 정밀하게 조정 (연한 민트색 배경의 안내 박스 추가).

### 5. Stage 2. 시나리오 시뮬레이터 (Chatbot)
- 5가지의 핵심 윤리 딜레마 시나리오를 정의하고, 사용자 선택에 따라 다른 상황이 시작되도록 설정.
  - *시나리오 예시: AI 채용 편향 (박준혁), 내부 코드 GPT 입력 (이서준) 등*
- `dotenv`를 통해 [.env] 파일에 있는 `OPENAI_API_KEY`를 로드하여 세션에 할당.
- **OpenAI API 연동 (`gpt-4o-mini`)**: [persona.md]의 구조(T1~T5 단계별 강경함 수준 변화)를 프롬프트(`sys_prompt`)로 변환하여 적용. 페르소나는 JSON 형태로 `stage`, `message`, `result` 값을 반환하도록 강제.
- 5턴 제한 이내에 논리적으로 페르소나를 완전히 설득(`T5` 도달 또는 `result="success"`)하거나, 실패할 경우 각각 성공/실패 화면 렌더링 및 재도전 옵션 부여.

### 6. Stage 3. 만족도 테스트 및 AI 분석 결과 (Satisfaction & Feedback)
- 시뮬레이션 종료 후 5개의 객관식 슬라이더와 1개의 주관식 피드백으로 구성된 만족도 폼 UI 구현.
- 사용자가 제출을 완료하면, 사용자가 앞선 시뮬레이터에서 나눈 **전체 대화 기록(history)**과 **설문 결과**를 모아 한 번 더 GPT API를 호출하여 분석.
- [score.md] 룰에 따라 다음 세 가지 핵심 정보를 추출하여 JSON으로 렌더링:
  1. **결정적 한 마디**: 사용자가 했던 말 중 가장 주요했던 인용구(`quote`)와 그 이유(`reason`).
  2. **설득 스타일 분석**: 6가지 유형(증거 제시형, 관계 기반형 등) 중 하나를 도출(`persuasion_type`).
  3. **총평 피드백**: `score_comment`.

### 7. 관리자 대시보드 (HR Dashboard)
- [hrdashboard.md]문서를 기반으로 "사용자 뷰"와 완전히 분리된 HR 임원용 대시보드 구현.
- 3가지 주요 KPI 표출 (전체 이수율 68%, 평균 만족도 4.2, 인기 시나리오).
- 팀별 이수 현황을 텍스트 기반 진행바(unicode blocks)와 결합하여 시각화.
- 사용자들이 Stage 3에서 분석받은 6가지 '설득 유형 분포'를 커스텀 CSS의 코닉 그레이디언트(`conic-gradient`) 비율 차트(도넛 형태)를 통해 가시적으로 표시.

## 🛠 버그 픽스 및 개선 사항
1. **내비게이션 인덴테이션(Indentation) 버그 픽스**:
   - 관리자 대시보드 통합 과정에서 파이썬 들여쓰기 오류로 인하여 일반 사용자가 0단계(프로그램 소개) 화면을 보지 못하는 문제 발생.
   - 렌더링 분기 로직(if/else)을 재조정하여 정상적으로 Intro 뷰에 진입하도록 수정.
2. **동영상 순차 시청 플로우 상세 적용**:
   - 사용자의 커스텀 오더("AI 윤리 개념" ~ "AI 사용 명시") 피드백을 반영하여 배열 구조를 재배치.
   - 현재 단계 동적 타이틀, UI 안내 메시지, 버튼 활성화를 와이어프레임과 동일한 형태로 고도화.

## 결론 및 현재 상태
현재 [app.py]는 기획된 5개의 시나리오와 교육 플로우, 대시보드를 100% 정상적으로 동작시키는 프로토타입 형태로 완성되었습니다. API 통신 안정성 구조와 커스텀 컴포넌트 아키텍처가 모두 수립되어 정상 구동 중입니다.
